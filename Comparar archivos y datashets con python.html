Comparar archivos y datashets con python (Pandas, MD5 y SHA1)¶

Cuando tenemos un conjunto de datos pequeo, una de las maneras ms simples de comparar dos datashets es con excel o similar, y comparar las dos hojas con esta sencilla instruccin en cada casilla (en la casilla A1, que tomaremos como referencia):

=Sheet1!A1=Sheet2!A1

Esta instruccin realizar una comparacin booleana entra las casillas A1 de las hojas llamadas "Sheet1" y "Sheet2" y devolver True en caso de que sean iguales y False en contrario. Sin embargo, con volumenes de datos muy grandes, trabajar con excel resulta costoso o impracticable. En situaciones como esta se utilizan herramientas alternativas, como R o python. En este notebook se presentan diversas maneras de comparar archivos y datashets en python, con una explicacin guiada de cada mtodo as como una implementacin ejecutable en python de cada uno.

Comparando datos utilizando el algoritmo MD5 CHecksum ¶

El algoritmo MD5 Hash es parte de la librera "hashlib" de python, acepta una secuencia de bytes y devuelve un valor hash de 128 bits. Se suele utilizar para comprobar la integridad de los datos, realizar comprobaciones o control de versiones. La documentacin se puede consultar en este enlace. Funciones principales:

  • encode (): Convierte el string en bytes para que sea aceptable por la funcin hash.
  • digest (): Devuelve los datos codificados en formato byte.
  • hexdigest (): devuelve los datos codificados en formato hexadecimal.

NOTA IMPORTANTE: es libreria slo funciona con strings, bytes os.PathLike object (argumentos pasados por la shell del sistema), luego no acepta dataframes o series por jupyter notebook. Sin embargo, si es capaz de comparar datashets si los pasamos por shell, siendo md5.py el script en python del cdigo que se presenta ms abajo y se puede encontrar en el repositorio:

$ python md5.py archivo1 archivo2

$ python md5(v2).py archivo1 archivo2

La versin v2 incluye un main(), pero el cdigo es el mismo.

El siguiente cdigo muestra el funcionamiento del MD5 hash que acepta bytes y devuelve la salida como bytes.

A continuacin se incluye el cdigo del script md5.py que comparar dos archivos s lo llamamos por shell, ya que como se ha indicado anteriormente, esta librera slo funciona con strings, bytes os.PathLike object (argumentos pasados por la shell del sistema), luego no acepta dataframes o series por jupyter notebook. Sin embargo, si es capaz de comparar datashets si los pasamos por shell, siendo md5.py el script en python del cdigo que se presenta ms abajo y se puede encontrar en el repositorio:

# script de md5(v2).py
# Comparar si dos archivos son iguales mediante MD5 Checksum (V2 con main)

# librerias
#import hashlib
import hashlib, sys

# paso de ficheros por argumento de shell
files = [sys.argv[1], sys.argv[2]] 
def md5(fname):
    md5hash = hashlib.md5()
    
    # Abriremos el fichero linea a linea, por cuestiones de memoria.
    with open(fname) as handle:
        for line in handle:
            md5hash.update(line.encode('utf-8'))
    return(md5hash.hexdigest())

def main():
    print('Comparando archivos',files[0],' y ',files[1], " usando MD5.")

    if md5(files[0]) == md5(files[1]):
        print('Coinciden.')
    else:
        print('No coinciden.')


if __name__ == "__main__":
    main()

Comparando datos utilizando el algoritmo SHA1 ¶

El algoritmo SHA1 es otro algoritmo hexadecimal que convertir el contenido de nuestro archivo en un string. Es el mismo algoritmo que Bitcoin usa para su proceso de minera blockchain. Es parte de la librera hashlib, y se utiliza exactamente igual que md5. Este es el contenido del script sha1.py el script que se puede encontrar en el repositorio:

$ python sha1.py archivo1 archivo2
